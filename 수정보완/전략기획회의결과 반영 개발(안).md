## ADR (Architecture Decision Record)

| 항목 | 내용 |
|------|------|
| **Decision** | `causal_chain_log` 별도 테이블 대신 `pet_status_events.metadata.hook_sentence` 필드 활용 |
| **Drivers** | 프롬프트 토큰 절약(~1500→~200), "lost in the middle" 방지, 기존 3계층 인과관계 시스템과 자연 통합 |
| **Alternatives** | (A) 별도 `causal_chain_log` 테이블 — 조인 비용 + 프롬프트 비대화 / (B) `firstSentence` 유지 — 훅이 아닌 도입부 저장으로 인과관계 단절 |
| **Why Chosen** | 기존 `metadata` JSONB 필드에 1개 키 추가로 스키마 변경 최소화. `lastSentence`가 system prompt의 "END WITH A HOOK" 규칙과 정합 |
| **Consequences** | 기존 `firstSentence` 기반 매핑 3곳 동시 수정 필요. 전환 기간 구/신 데이터 혼재 → `lastSentence || firstSentence` fallback 패턴 필수 |
| **Follow-ups** | Paddle 승인 후 `lemon_squeezy_*` 환경변수 동시 정리 / `npc_state` 테이블 존재 여부 확인 필요 |

**핵심 로직**:
```typescript
// lib/utils/sentence-parser.ts
export function extractLastSentence(text: string, lang: 'en' | 'ko' | 'ja'): string {
  const delimiters = /[.。!！?？]+/
  const sentences = text.split(delimiters).map(s => s.trim()).filter(Boolean)
  return sentences[sentences.length - 1] || sentences[0] || text.slice(0, 80)
}

// prompt-builder.ts yesterdayHook 수정
const yesterdayHook = recentEvents[0]?.metadata?.hook_sentence
  || recentEvents[0]?.firstSentence  // fallback for old data
  || ''
```

**수용 기준**:

- [ ] 일본어: `。` 기반 마지막 문장 파싱 정확
- [ ] 영어: `.!?` 기반 정확

```typescript
export function calculateStreak(letters: { created_at: string }[]): number {
  // 날짜별 그룹화 후 연속 일수 계산
  const dates = [...new Set(letters.map(l => l.created_at.slice(0, 10)))]
    .sort().reverse()
  let streak = 0
  let prev = new Date()
  for (const date of dates) {
    const diff = Math.round((prev.getTime() - new Date(date).getTime()) / 86400000)
    if (diff <= 1) { streak++; prev = new Date(date) }
    else break
  }
  return streak
}
```
