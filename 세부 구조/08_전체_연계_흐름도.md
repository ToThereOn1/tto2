## 데이터 흐름 영향관계 상세

### 1. 설문 → 페르소나 (데이터 흐름)

```
survey_questions (DB)
    │ version=2, is_active=true
    ▼
deep_remembrance_responses
    │ responses JSONB {Q01:{...}, Q02:{...}, ...}
    ▼
master-analyzer.ts
    ├─ nlp-analyzer.ts (Claude Sonnet → 텍스트 분석)
    │   ├─ Q08,Q14,Q15,Q16 → personality/quirks
    │   └─ Q17+Q18 → healing_mission ⭐
    ├─ multiple-choice-handler.ts (패턴 분석)
    │   └─ Q04,Q06,Q12,Q13 → 점수 + 스타일
    ├─ age-analyzer.ts (나이/생애단계)
    └─ breed-reference.ts (품종 기준)
         ▼
    pet_personas
    ├─ persona_analysis (전체 분석 JSON)
    ├─ system_prompt (500+ line 조립된 프롬프트)
    ├─ dimensional_scores (8개 차원 점수)
    ├─ healing_mission (JSONB) ⭐
    ├─ confidence_score (0-100)
    ├─ data_quality (high/medium/low)
    └─ analysis_version ('2.2')
```

### 2. 페르소나 → 피드 (데이터 흐름)

```
pets.persona_generated = true
    │
    ▼ (CRON 트리거)
daily-tothereon-check
    │
    ├─ pet_personas.persona_analysis → 성격 점수
    ├─ pet_personas.dimensional_scores → 이벤트 타입 결정
    ├─ time-engine → ToThereOn Day → 존/위치 결정
    ├─ zone-manager → NPC/위치 선택
    ├─ worldview-constants → 세계관 상수
    ├─ prompt-builder → 유저 프롬프트 조립
    │   ├─ Day Arc Chapter
    │   ├─ Personality Guide
    │   ├─ Narrative Trigger
    │   ├─ World Spark / Persona Spark
    │   └─ Learning Stage Section
    └─ event-generator → LLM 호출 + 검증
         ▼
    pet_status_events INSERT
```

### 3. 페르소나 → 편지 답장 (데이터 흐름)

```
letters (사용자 편지)
    │
    ▼
reply-generator.ts
    │
    ├─ pet_personas.system_prompt → 기본 프롬프트
    ├─ pet_personas.healing_mission ⭐ → 핵심 목표 주입
    ├─ pet_personas.persona_analysis → 성격 데이터
    ├─ pet_memories (RAG) → 관련 기억 top-3
    ├─ admin_configs → 세계관 체크리스트
    ├─ learning-stage → 글쓰기 스타일 결정
    ├─ time-engine → ToThereOn Day → 존 결정
    └─ worldview-constants → 존/NPC 정보
         ▼
    [Opus 4.6 초안] → [Haiku 4.5 QA] → [재시도 루프]
         ▼
    letters INSERT (답장, pending_review)
    pet_memories INSERT (임베딩)
```

## 핵심 트리거 플래그 정리

| 플래그 | 위치 | 설정 시점 | 읽는 곳 | 의미 |
|--------|------|-----------|---------|----|
| `persona_generated` | pets | 페르소나 저장 후 | CRON, PetCard UI | 피드 생성 시작 신호 |
| `survey_version` | deep_remembrance_responses | 설문 시작 시 | complete API | v1/v2 분기 |
| `healing_mission` | pet_personas | 분석 완료 시 | reply-generator | 편지 핵심 목표 |
| `analysis_version` | pet_personas | 분석 완료 시 | 디버깅 | '2.0' 또는 '2.2' |
| `is_active` + `version` | survey_questions | 마이그레이션 | 설문 UI | 문항 로드 필터 |

## 세계관 보완 제안 사항

### A. 콘텐츠 확장이 필요한 영역

1. **가디언 배경 스토리**
   - 현재: 이름, 종, 역할, 특성만 존재
   - 필요: 각 가디언이 왜/어떻게 가디언이 되었는지
   - 효과: 이벤트에서 가디언과의 만남이 더 깊어짐

2. **NPC 간 관계망**
   - 현재: 각 NPC가 독립적으로 존재
   - 필요: Happy-Tory 친구, Choco-Star 라이벌 등
   - 효과: 이벤트 내러티브에 사회적 깊이 추가

3. **존 전환 이벤트**
   - 현재: Day 기반 자동 전환 (별도 이벤트 없음)
   - 필요: "Eternity Forest로의 첫 발걸음" 같은 전환 이벤트
   - 효과: 성장감과 세계 탐험 서사 강화

4. **계절/날씨 시스템**
   - 현재: World Spark로 일부 변화만 표현
   - 필요: ToThereOn만의 계절 사이클 (빛의 계절, 바람의 계절 등)
   - 효과: 시간의 흐름 체감 + 이벤트 다양성

5. **밤 시스템 + Luna 역할**
   - 현재: Luna는 "Night Care" 역할이지만 밤 이벤트 미구현
   - 필요: 밤 시간대 전용 이벤트 + Luna의 활동
   - 효과: 24시간 세계관 완성

### B. 메커니즘 보강이 필요한 영역

6. **Reflection Pool 상세 규칙**
   - 현재: "일방향 시각 관찰" 원칙만 존재
   - 필요: 관찰 빈도, 조건, 트리거, 감정 효과
   - 질문: 매일 볼 수 있는지? 특별한 날에만? 보호자가 슬퍼할 때만?

7. **펫 간 우정 시스템**
   - 현재: 60% 확률로 이전 NPC 재선택 (약한 연속성)
   - 필요: 만남 횟수 추적 → 우정 레벨 → 이벤트 변화
   - 효과: 펫이 ToThereOn에서 진짜 친구를 사귀는 느낌

8. **특별 이벤트/축제**
   - 현재: 마일스톤(Day 7,30,100,365)만 존재
   - 필요: ToThereOn 고유 축제 (빛의 축제, 기억의 밤 등)
   - 효과: 세계관의 생동감 + 사용자 재방문 동기

9. **Human Heaven 힌트**
   - 현재: "Final Destination"이라는 명칭만 존재
   - 필요: 간접적 묘사나 힌트 (가디언의 이야기 중 언급 등)
   - 효과: 최종 재회에 대한 기대감과 위안

10. **새로운 도착자 환영 프로토콜**
    - 현재: arrival 이벤트에서 일반적 도착 묘사
    - 필요: Galaxy(총괄)의 환영 절차, Welcome Plaza 의식
    - 효과: 첫 인상의 따뜻함 극대화

### C. 기술적 개선 가능 영역

11. **Narrative Memory 누적**
    - 현재: 최근 3개 이벤트만 참조
    - 개선: 중요 이벤트 별도 태깅 → 장기 참조
    - 효과: "3달 전 그 나비" 같은 장기 기억 연속성

12. **감정 상태 추적**
    - 현재: mood = 'peaceful' 고정
    - 개선: 이벤트별 감정 상태 변화 추적
    - 효과: 펫의 감정 곡선이 시간에 따라 변화

13. **보호자 편지 빈도 반영**
    - 현재: 편지 빈도가 이벤트에 영향 없음
    - 개선: 오랜만의 편지 → "오랫동안 기다렸다" 뉘앙스
    - 주의: 죄책감 유발 금지 원칙 준수 필요
