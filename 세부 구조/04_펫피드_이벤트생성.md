# 04. 펫 피드 이벤트 생성 시스템

## 개요

매일 CRON Job이 실행되어, 페르소나가 생성된 펫들에 대해
AI 기반 일일 상태 이벤트(피드)를 생성합니다.

**트리거**: `GET /api/cron/daily-tothereon-check`
**모델**: Claude Sonnet 4.6
**Temperature**: 0.8 (내부 기본값)
**최대 토큰**: 1000
**재시도**: 최대 3회 + 검증 루프

---

## CRON Job 플로우

### 1. 인증 및 초기화

```
GET /api/cron/daily-tothereon-check
Authorization: Bearer ${CRON_SECRET}

├─ 베어러 토큰 검증
├─ 글로벌 Day 계산 (첫 번째 펫의 passed_date 기준)
├─ 처리 상태 체크 (world_processing_state 테이블)
└─ 이미 완료된 날이면 → 조기 반환
```

### 2. 펫 선별

```sql
SELECT * FROM pets
WHERE persona_generated = true
```

### 3. 적격성 체크 (각 펫별)

```
Check 1: currentDay >= 1 (ToThereOn Day 최소 1일)
Check 2: 오늘 이벤트 이미 존재하는지
         → pet_status_events WHERE pet_id = ? AND tothereon_day = ?
Check 3: 피드 주기 게이트 (구독 티어별)
         └─ daysSinceLast < feedInterval → 건너뜀
```

### 4. 피드 주기

| 티어 | 주기 | 빈도 |
|------|------|------|
| Free | 7 Earth days | 주 1회 |
| Basic | 3 Earth days | 주 2회 |
| Premium | 3 Earth days | 주 2회 |

### 5. 청크 처리

- **최대 펫 수/실행**: 20마리 (CHUNK_SIZE)
- **최대 실행 시간**: 300초 (Vercel Pro)
- **처리 상태**: `world_processing_state`에 진행 기록

```
이미 처리된 펫 ID → Set으로 필터링
미처리 펫 중 20마리만 선택 (chunk)
처리 완료 후 상태 업데이트
```

## 이벤트 타입 결정 로직

### 우선순위 기반 결정

```typescript
function determineEventType(currentDay, hasRecentLetter, scores, ...): EventType {

  // Priority 1: 편지 반응 (168h+ 게이트 통과 시)
  if (hasRecentLetter && letterReplyVisible) → 'letter_response'

  // Priority 2: Day 0 → 도착
  if (currentDay === 0) → 'arrival'

  // Priority 3: 학습 스케줄 (Day 0~8 구간)
  if (learningStage !== 'completed') {
    if (currentDay === obsDay)      → 'learning_observation'
    if (currentDay === lessonDay)    → 'first_lesson'
    if (currentDay === practiceDay)  → 'practice_session'
  }

  // Priority 4: 마일스톤
  if ([7, 30, 100, 365].includes(currentDay)) → 'milestone'

  // Priority 5: 성격 기반 확률 분배
  if (social_energy > 70 && rand < 0.3)   → 'npc_interaction'
  if (curiosity_drive > 70 && rand < 0.4) → 'exploration'

  // 기본 가중 랜덤
  dice < 0.4  → 'daily_routine'
  dice < 0.65 → 'exploration'
  else        → 'npc_interaction'
}
```

### 이벤트 타입 목록

| 타입 | 설명 |
|------|------|
| `arrival` | Day 0: 펫이 ToThereOn에 도착 |
| `exploration` | 방랑, 발견 |
| `npc_interaction` | NPC와 만남 |
| `daily_routine` | 일상 활동 |
| `letter_response` | 보호자 편지에 반응 (168h 게이트) |
| `milestone` | Day 7, 30, 100, 365 |
| `learning_observation` | 멀리서 관찰 (초기) |
| `first_lesson` | 선생님 NPC 근처 (초기) |
| `practice_session` | 혼자 반복 시도 (초기) |

### 학습 스케줄

| 속도 | 관찰일 | 수업일 | 연습일 |
|------|--------|--------|--------|
| fast (IQ ≥ 75) | Day 2 | Day 3 | Day 4 |
| medium (IQ ≥ 50) | Day 3 | Day 5 | Day 7 |
| slow (IQ < 50) | Day 4 | Day 6 | Day 8 |

### 지능 점수 계산

```
base = 50
+ (empathy_sensitivity - 50) × 0.2
+ (curiosity_drive - 50) × 0.2
+ (social_energy - 50) × 0.1
= range: 10~100
```

## LLM 프롬프트 구조

### System Prompt (전체)

```
You are the only witness to what happens in ToThereOn World — a transit station
between Earth and the beyond, where beloved pets live their days until reunion.

YOUR MISSION:
Write one real moment from today. Specific enough, honest enough,
that the guardian reading it feels: "That's my pet. They're actually there."

━━━ THREE CORE RULES ━━━

RULE 1 — CONTINUE THE STORY
Reference at least one thing from recent events.
The story must feel like it is in the middle of something.

RULE 2 — LET THE BODY SPEAK
✓ "Three hesitant steps before approaching"
✓ "Returned to the same stone for the third time this week"
✗ "felt nervous", "was happy", "she realized" → FORBIDDEN

RULE 3 — THIS ANIMAL SPECIFICALLY
Include one physical or behavioral detail that belongs only to them.

━━━ FORBIDDEN LIST ━━━
1. Emotion naming: happy, sad, joyful, lonely, grateful, serene...
2. Interior access: felt, thought, realized, wondered, remembered...
3. Causal attribution: because of the letter, thanks to...
4. Melodrama: crying, sobbing, quiet ache, rainbow bridge...
5. Generic filler: had fun, enjoyed the day...
6. ⛔ GUARDIAN'S REAL WORLD — ABSOLUTE PROHIBITION
   NEVER write about what the guardian is doing.
   Guardian exists ONLY as: a scent, a learned habit, a posture.

━━━ NARRATOR VOICE ━━━
Warm, affectionate narrator — like a children's nature documentary.
✓ First sentence: warm overview setting emotional temperature.
✗ No dry description. No sentimental melodrama.

LANGUAGE: English ONLY.
LENGTH: 4–6 sentences. Third person, past tense.
OUTPUT: Only the text. No title. No headers.
```

## World Spark 시스템

각 존에 7-9개의 랜덤 세계 상태 디테일. 빈 문자열도 포함 (평범한 날).

**예시 (Crystal Meadow)**:
- "The crystal flowers are mid-bloom — their reflections scatter light"
- "Several recently-arrived pets are near the butterfly garden"
- "Kitsune passed through earlier — tracks suggest considerable speed"
- "" (평범한 날)

**예시 (Eternity Forest)**:
- "The blue moss is particularly bright today — it glows even in daylight"
- "There is a low sound from somewhere deeper in the forest"
- "Several pine cones have fallen, arranged in a way that does not look accidental"

## Persona Spark 시스템

성격 점수 기반 행동 변이 + 모순(contradiction) 생성.

```
curiosity_drive > 80:
  "investigated something entirely unrelated to where they were supposed to be"

playfulness > 80:
  "something started as play and became something else"

anxiety > 70:
  "took the long way, or stopped and started twice before committing"

social_energy < 30:
  "actively found a quieter spot when the usual place got crowded"

Contradiction (30% 확률):
  social_energy < 40: "the one who usually keeps distance made the first move"
  playfulness < 30: "something made the calm one move quickly"
  anxiety > 60: "the cautious one went somewhere new first, without hesitating"
```

## DB 저장

```sql
INSERT INTO pet_status_events (
  pet_id,
  tothereon_day,
  event_type,        -- 'arrival'|'exploration'|'npc_interaction'|...
  event_title,       -- 'Day X'
  event_description, -- LLM 생성 텍스트

  event_language,    -- 감지된 언어
  zone,              -- 현재 존 ID
  location,          -- 위치 ID
  npc_involved,      -- NPC 이름 (nullable)
  is_learning_event, -- boolean
  learning_stage,    -- 학습 단계 이름
  metadata           -- { trigger, location_name, npc_name, time_of_day,
                    --   validation_passed, attempts }
)
```

## 프라이버시 및 안전

```
1. allow_world_interaction = false → 다른 펫과 상호작용 차단
2. 근처 펫 하드캡: 최대 2마리 (DEFENSE FIX 4)
3. 근처 펫 가중치: 20%만 (80%는 본인 이야기)
4. 상호작용 파트너 중복 처리 방지
5. 처리 상태 추적으로 이중 처리 방지
```
