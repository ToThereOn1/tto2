## 1. 서비스 개요

### 1-1. 목적과 핵심 가치

ToThereOn은 반려동물을 잃은 보호자의 그리프(grief)를 AI를 통해 케어하는 SaaS 서비스이다.

**핵심 가치: "연결의 지속"**

죽음 이후에도 반려동물과의 유대가 끊어지지 않았다는 감각을 제공한다. 무지개다리 너머의 중계 세계(ToThereOn World)에서 반려동물이 실제로 생활하며, 보호자에게 편지를 보내고, 보호자는 답장을 쓸 수 있다.

서비스가 제공하는 것은 "위로"가 아니라 **"목격"** 이다. ToThereOn World의 관찰자(narrator)는 반려동물의 일상을 보고하며, 보호자는 그 보고를 통해 반려동물이 안전하고 행복하게 지내고 있다는 것을 알게 된다.

세계관의 핵심 법칙 3가지 (`lib/world-bible.ts` UNIVERSE_LAWS):
1. **경유 의무(Transit Mandatory)**: 지구에 보호자가 있는 모든 반려동물은 ToThereOn World를 반드시 거친다
2. **재회 보장(Reunion Guaranteed)**: 100% 재회 — 확률이 아니라 시간의 문제
3. **함께 떠남(Departure Together)**: 재회 후 3-7일을 함께 보내고, 둘이 함께 Human Heaven으로 떠난다

### 1-2. 타겟 유저

| 구분 | 설명 |
|------|------|
| **Primary** | 반려동물(개/고양이)을 잃은 보호자 (pet loss grief 상태) |
| **Secondary** | 반려동물의 죽음을 앞두고 있는 보호자 (anticipatory grief) |
| **지역** | 영어권 우선 (한국어, 일본어 지원 준비됨 — `lib/language-detector.ts`) |
| **연령** | 25-55세, 반려동물에 깊은 감정적 유대를 가진 디지털 사용자 |

### 1-3. 핵심 차별화 요소

**1. 12-Step Persona Engine**
21문항 Deep Remembrance 설문 + Claude Sonnet NLP 분석 + 품종 블렌딩으로 "이 반려동물만의" 성격을 정밀 재현한다. 단순한 챗봇이 아니라, 보호자만 아는 고유한 행동 패턴과 성격 특성을 AI가 학습한다.

**2. Healing Mission 시스템**
Q17("듣고 싶은 말") + Q18("전하고 싶은 말")에서 추출한 `core_desire`, `guilt_points`, `closure_level`이 모든 편지 생성의 최우선 목표로 주입된다 (`pet_personas.healing_mission`). 보호자의 미해결 감정을 편지가 점진적으로 다룬다.

**3. 7-Stage Learning System**
편지 글쓰기가 Day 0부터 점진적으로 진화한다 (`lib/learning-stage.ts`, `lib/time-constants.ts`):
- Stage 1 (Day 0-3): typo rate 80%, Gaegu 폰트, wobble +/-5deg
- Stage 5 (Day 12-66): typo rate 5%, Nanum Pen Script
- Stage 7 (Day 167+): typo rate 0%, Nanum Myeongjo, 유려한 문체
시각적으로 반려동물의 성장이 보인다.

**4. MMORPG 세계관**
7명의 캐논 NPC(`lib/world-bible.ts` CANON_NPCS), 10명의 동반자 NPC(`COMPANION_NPCS`), 8개 캐논 장소(`CANON_LOCATIONS`), 고유 시간 법칙(3:1), Dual Moon 시스템, Reflection Pool 규칙 등 살아있는 세계.

**5. RAG Memory**
편지 내용이 OpenAI text-embedding-3-small로 벡터화되어 축적된다 (`lib/memory-store.ts`). 시간이 갈수록 반려동물이 보호자의 기억을 더 깊이 반영하는 답장을 쓴다.

## 2. 작동 방식

### 2-1. 사용자 여정 전체 흐름

```
[1] 가입 (Supabase Auth - Google/Email)
     |
[2] 펫 등록 (/dashboard/pets/new)
     |- PetRegistrationForm: 이름, 종, 품종, 관계, 사진 1-5장
     |- pet_limit 체크 (Free/Basic: 1마리)
     |- Supabase Storage (pet-photos 버킷) + pets INSERT
     |
[3] Deep Remembrance 설문 (/dashboard/pets/[petId]/remembrance)
     |- 21문항, 5 Phase (The Door -> Living Room -> Quiet Room -> Letter -> Light)
     |- 5초마다 자동저장 -> deep_remembrance_responses (JSONB)
     |- Q17(듣고 싶은 말), Q18(전하고 싶은 말) = Healing Mission 핵심 소스
     |- 완료 시 POST /api/deep-remembrance/complete
     |
[4] 페르소나 생성 (백그라운드, 비동기)
     |- after() 비동기: master-analyzer.ts 12-Step Pipeline
     |- Visual DNA: GPT-4o Vision -> pets.visual_description (병렬)
     |- NLP 분석: Claude Sonnet 4.6 (Q08, Q14, Q15, Q16, Q17, Q18)
     |- 출력: pet_personas UPSERT (persona_analysis, system_prompt, healing_mission, dimensional_scores)
     |- pets.persona_generated = TRUE -> CRON 활성화 트리거
     |
[5] 펫 피드 수신 (CRON: /api/cron/daily-tothereon-check)
     |- 피드 주기 게이트: Free 7일 / Basic 3일 (lib/time-constants.ts FEED_FREQUENCY)
     |- 이벤트 타입 결정 (arrival -> learning -> milestone -> personality-based)
     |- Claude Sonnet 4.6으로 1인칭 저널 생성 (80-130 words)
     |- 검증: 금지어 + 문장수 + 단어수 150 cap
     |- pet_status_events INSERT
     |
[6] 편지 주고받기
     |- 보호자 편지: POST /api/letters/send (quota 체크 -> letters INSERT)
     |- 편지 타임라인 (lib/time-constants.ts LETTER_PIPELINE):
     |    sent(0h) -> crossing(72h) -> arrived(72h) -> reading(96h) -> writing(120h) -> replied(168h)
     |- AI 답장: generateLetterReply() -> Claude Opus 4.6 (temp 0.85-1.0)
     |- QA 검수: Claude Haiku 4.5 (score >= 80, 최대 3회 재시도)
     |- Admin 승인 -> delivered
     |- 양방향 RAG: 보낸 편지/받은 답장 모두 OpenAI embedding 저장
```

### 2-2. 주요 시스템 컴포넌트

#### 타임 엔진
- `TIME_RATIO = 3` (지구 3일 = ToThereOn 1일) — `lib/time-constants.ts:7`
- `lib/time-engine.ts`: startDate 기준 currentDay 계산, 존 매핑
- 편지 배달 파이프라인: 6단계 상태 변화 (0h -> 72h -> 96h -> 120h -> 168h)

#### 학습 단계 시스템 (7-Stage)
`lib/learning-stage.ts` + `lib/time-constants.ts`에서 정의:

| Stage | Day 범위 | 이름 | Typo Rate | Vocab Richness | 폰트 |
|-------|---------|------|-----------|----------------|------|
| 1 | 0-3 | First Light | 0.80 | 0.10 | Gaegu |
| 2 | 4-6 | Budding Voice | 0.60 | 0.25 | Gaegu |
| 3 | 7-9 | Growing Words | 0.35 | 0.45 | Nanum Pen |
| 4 | 10-11 | Finding Flow | 0.15 | 0.65 | Nanum Pen |
| 5 | 12-66 | Clear Expression | 0.05 | 0.85 | Nanum Pen |
| 6 | 67-166 | Poetic Soul | 0.00 | 0.95 | Nanum Myeongjo |
| 7 | 167+ | Timeless Voice | 0.00 | 1.00 | Nanum Myeongjo |

World Bible 규칙: 모든 답장 길이는 보호자 편지의 ~130% (`STAGE_LENGTH_MULTIPLIER` 전 단계 1.3 통일). "저편의 그리움은 언제나 조금 더 크다."

#### 메모리 RAG
- `lib/memory-store.ts`: OpenAI text-embedding-3-small
- 편지 전송/수신 시 임베딩 저장 (`pet_memories` 테이블)
- 답장 생성 시 top-3 관련 기억 검색 (`search_pet_memories` RPC)

#### 세계관 구성 요소
- **캐논 장소 8곳** (`lib/world-bible.ts` CANON_LOCATIONS): 도착의 문, 수로, 콩선생 모래밭, 꼬리상인 시장, 빵집 처마 아래, 꽃밭, 두 달 언덕, 개인 자리
- **캐논 NPC 7명** (`lib/world-bible.ts` CANON_NPCS): Granny Shell(거북, 도착 맞이), Professor Clover(토끼, 글쓰기 선생), Pip(편지 배달부), Old Finn(여우, 이야기꾼), Bun & Bun(토끼 자매, 빵집), Digby(두더지, 탐험), Lune(흰 사슴, 마일스톤)
- **동반자 NPC 10명** (`lib/world-bible.ts` COMPANION_NPCS): Happy, Choco, Tory, Cloud, Lightning, Star, Mong, Ruby, Wind, Bokshil
- **Dual Moon 시스템**: Near Moon(지구 반사), Far Moon(ToThereOn 반사) — `DUAL_MOON`
- **Reflection Pool 규칙**: 보호자를 "볼 수 없다" — 오직 온기(warmth)만 느낀다 — `REFLECTION_POOL_RULES`

### 2-3. AI 파이프라인

4개 LLM + 2개 OpenAI 모델을 사용한다 (`lib/ai-config.ts`):

| 역할 | 모델 | Temperature | 용도 | 파일 |
|------|------|------------|------|------|
| 설문 NLP 분석 | Claude Sonnet 4.6 | 0.2-0.3 | 주관식 성격/감정 추출, Healing Mission | `lib/master-analyzer.ts` |
| 피드 이벤트 생성 | Claude Sonnet 4.6 | 기본값 | 1인칭 저널 생성 (80-130 words) | `lib/event-generator.ts` |
| 편지 답장 생성 | Claude Opus 4.6 | 0.85-1.0 | 감정적 편지 (학습단계별 스타일) | `lib/reply-generator.ts` |
| QA 자동검수 | Claude Haiku 4.5 | 0 | 세계관/안전/페르소나 검증 (80점 통과) | `lib/reply-generator.ts:640` |
| Visual DNA | GPT-4o | 0.3 | 사진 -> 외형 묘사 | `lib/master-analyzer.ts` |
| RAG 임베딩 | OpenAI text-embedding-3-small | - | 편지 벡터 저장/검색 | `lib/memory-store.ts` |

### 2-4. 핵심 기술 구조

```
[사용자 입력] deep_remembrance_responses (JSONB)
       |
       v  master-analyzer.ts (12-Step Pipeline)
  pet_personas
    |- persona_analysis (전체 분석 JSON)
    |- system_prompt (500+ line)
    |- healing_mission (core_desire, guilt_points, closure_level)
    |- dimensional_scores (8차원)
       |
       |--- [CRON] ----> pet_status_events (피드)
       |    event-generator.ts + prompt-builder.ts
       |    zone-manager.ts + learning-stage.ts
       |    world-bible.ts + npc-service.ts + witness-service.ts
       |
       |--- [USER LETTER] ----> letters (답장)
            reply-generator.ts (Opus 4.6)
            memory-store.ts (RAG)
            learning-stage.ts + writing-mastery.ts
```

**방어 코딩 현황:**
- CRON 청크 처리: `CHUNK_SIZE=20`, `maxDuration=300`
- 이중 처리 방지: `processedByInteraction` Set
- 진행 추적: `world_processing_state` 테이블
- Admin 검수 파이프라인: Haiku QA + 3회 재시도 + `pending_review` 상태

#### 차원 1: 핵심 AI 품질 (편지 + 피드의 진정성) — 19 / 25

**근거:**
- **피드 품질**: v15 최고점 87.1/100, 현재 85.8-86.9 안정권. "보호자연결" 차원 23.7/25로 강점. 그러나 "서사인과" 19.9/25, "세계관체감" 20.0/25로 약점이 존재한다. (`lib/prompt-builder.ts` — 프롬프트 지시 간 우선순위 모호성이 LLM을 "안전한 중간 지대"로 수렴시킴)
- **편지 품질**: Claude Opus 4.6 + Haiku QA 3회 재시도 파이프라인이 견고하다 (`lib/reply-generator.ts`). Healing Mission 주입으로 보호자의 핵심 감정에 닿는 답장 가능.
- **감점 요인 (-6점)**:
  - 보호자 편지 echo 구조 부재: 보호자 편지가 user message로만 전달되고, system prompt에서 세계관 일상이 70% 지배. "내 편지를 읽었다"는 감각이 구조적으로 약함 (`lib/reply-generator.ts` buildSystemPrompt — 개선 계획 수립됨)
  - arrival 이벤트가 하드코딩 fallback: 첫 인상이 가장 중요한데 가장 낮은 품질. 정적 문자열 1개 반환 (`lib/event-generator.ts:684-698`)
  - EMOTIONAL_STATES 배열이 랜덤 선택 (`lib/reply-generator.ts:48-56`) — 편지 내용 기반 감정 파생이 아님
  - `reviewReply` 에러 시 `score: 100, passed: true` 반환 (`lib/reply-generator.ts:651-660`) — 네트워크 오류 시 미검수 편지가 완벽 점수로 Admin에게 전달됨

## 5. 개선 방향 (우선순위)

> **우선순위 재배열 근거 (Critic ITERATE 판정):**
> - 편지 echo 품질이 서비스 존재 이유 -> 1순위
> - arrival event가 첫 인상 결정 -> 1순위에 포함
> - 결제는 외부 블로킹(Paddle 심사) -> 별도 대기큐 분리
> - 리텐션이 결제보다 선행해야 함 (유저가 없으면 결제도 무의미)

### 1순위: 편지 echo 품질 + arrival event [기대 임팩트: CRITICAL]

**편지 echo 강화:**
- 보호자 편지 내용을 system prompt에도 구조적으로 주입 (60% echo / 30% 세계관 / 10% 성장)
- EMOTIONAL_STATES 랜덤 -> 보호자 편지 기반 감정 파생
- 학습 단계 오탈자 few-shot 예시 강화
- 관련 파일: `lib/reply-generator.ts` buildSystemPrompt()

**arrival event LLM 전환:**
- `generateArrivalEvent()` 정적 문자열 -> Claude Sonnet 4.6 동적 생성
- 펫 이름, 종, 품종, 성격(persona)을 반영한 개인화된 도착 이벤트
- Granny Shell(도착 맞이 NPC) 등장을 자연스럽게 포함
- 관련 파일: `lib/event-generator.ts:684-698`

**이유:** 편지가 서비스의 핵심 가치이며 유료 전환 동기이다. "내 편지를 읽었다"는 감각이 치유의 핵심. arrival event는 첫 인상이며, 현재 가장 낮은 품질.

### 2순위: Deep Remembrance UX 재설계 [기대 임팩트: HIGH]

- 입장 의식 (무지개다리 인트로 스크린)
- Phase 전환 의식 (5단계 색조/파티클 차별화)
- Q17 EmotionalBreather (감정 클라이맥스 전 숨 고르기)
- 무지개다리 탐색 완료 의식 (12-15초 애니메이션)
- Journey Path 진행 시각화 ("Step X / 21" 제거)
- 관련 파일: `components/remembrance/DeepRemembranceSurvey.tsx`

**이유:** 설문 이탈률이 서비스 전환율의 최대 병목. 첫 경험이 서비스의 "약속"을 설정.

### 3순위: 리텐션 루프 구축 [기대 임팩트: MEDIUM-HIGH]

- 편지 도착 알림 (이메일/웹 푸시)
- 마일스톤 이벤트 특별 알림 (Day 7, 30, 100)
- 존 전환 의식 이벤트
- 피드 없는 기간 "펫이 당신을 기다리고 있어요" 리마인더
- NPC 3중 진실 원천 통합 (H2 해소)

**이유:** 최초 전환 이후 장기 유지율이 SaaS 생존의 핵심. 현재 재방문 동기가 피드 주기에만 의존.

### 4순위: 피드 품질 88점 달성 [기대 임팩트: MEDIUM]

- 4개 구체 변경안 적용 (just_received echo, still_carrying 장소/신체, routine 심화 분기, NPC 두 박자)
- mood 하드코딩 제거 -> 이벤트 타입/성격 기반 동적 mood
- kitsune phantom NPC 제거 + NPC 활동 테이블을 COMPANION_NPCS 기반으로 재구성
- reviewReply 에러 시 `passed: false` + admin alert 전환
- 관련 파일: `lib/prompt-builder.ts`, `lib/universe/npc-service.ts`, `lib/reply-generator.ts:651-660`

**이유:** 피드가 무료 유저의 핵심 경험. 85 -> 88점은 작은 수치지만 보호자 체감에서 결정적 차이.

### 대기큐: 결제 게이트웨이 확보 [외부 블로킹]

- Paddle 심사 승인 즉시 통합 (플랜 파일 준비 완료: `docs/plans/2026-03-05-paddle-integration.md`)
- 거절 시 Polar.sh 백업 (한국 지원, MoR 방식)
- Lemon Squeezy 레거시 코드 정리
- **트리거**: Paddle 승인 통보 시 "paddle 계획 실행해줘"
- **상태**: 개발로 해결 불가, Jay 선행 작업 대기

**이유:** 결제 없이는 서비스 지속 불가하나, 외부 의존(Paddle 심사)이므로 대기큐로 분리. 내부 개선(1-4순위)과 병렬 진행.
