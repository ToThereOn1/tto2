# 05. 편지 답장 생성 시스템

## 개요

사용자가 펫에게 편지를 보내면, AI가 펫의 페르소나로
감정적인 답장 편지를 생성하는 시스템입니다.

**파일**: `lib/reply-generator.ts`
**초안 모델**: Claude Opus 4.6 (High tier)
**QA 모델**: Claude Haiku 4.5 (Low tier)
**재시도**: 최대 3회 (QA 통과까지)

---

## 편지 상태 파이프라인

| 단계 | 시간 (보낸 후) | 상태 | 설명 |
|------|---------------|------|------|
| 보냄 | 0h | `sent` | 사용자가 편지 전송 |
| 건너감 | 0-72h | `crossing_the_waterway` | 우주 수로를 건너는 중 |
| 도착 | 72h | `arrived_at_tothereon` | ToThereOn에 도착 |
| 읽는 중 | 96h | `reading_your_heart` | 펫이 편지를 읽음 |
| 쓰는 중 | 120h | `writing_reply` | AI 답장 생성 트리거 |
| 검토 중 | 120-168h | `pending_review` | 관리자 검토 대기 |
| 답장 완료 | 168h (7일) | `replied` | 사용자에게 답장 노출 |

## 답장 생성 플로우

```
사용자 편지 전송
    ↓
POST /api/letters/send
    ├─ quota 체크 (월별 편지 한도)
    ├─ letters INSERT (status: sent)
    └─ generateLetterReply() 트리거
         ↓
    [1] 데이터 로드
        ├─ Pet info + Persona profile
        ├─ pet_personas.healing_mission ⭐
        ├─ Admin configs (worldview + checklist)
        └─ pet_personas.system_prompt
         ↓
    [2] RAG 메모리 검색
        └─ retrieveRelevantMemories(petId, letterContent, top-3)
         ↓
    [3] 시간 + 학습 계산
        ├─ ToThereOn Day 계산
        ├─ 현재 존 결정
        ├─ 학습 단계 (typo_rate, vocab_richness)
        └─ 답장 길이 = 사용자 편지 길이 × stage multiplier
         ↓
    [4] 랜덤 요소 선택
        ├─ emotionalState (7가지 중 1개)
        └─ letterMode (5가지 중 1개 → temperature 결정)
         ↓
    [5] 시스템 프롬프트 조립
        └─ buildSystemPrompt(all parameters)
         ↓
    [6] 초안 생성 (Attempt 1)
        ├─ 모델: Claude Opus 4.6
        ├─ Temperature: letterMode.temperature (0.85~1.0)
        └─ Max Tokens: dynamic (1024~2048)
         ↓
    [7] QA 자동 검수 (Haiku)
        ├─ 세계관 준수
        ├─ 안전/윤리
        ├─ 페르소나 일관성
        ├─ 감정 품질
        └─ 점수: 0-100 (80+ 통과)
         ↓
    [8] 통과 시 → 저장
        실패 시 → 피드백 주입 후 재생성 (최대 3회)
         ↓
    [9] letters INSERT (status: pending_review)
        + 임베딩 저장 (RAG용)
```

## 시스템 프롬프트 전체 구조

```
[Language Rule] Write ONLY in English.

You are {petName}. {species}. You currently live in {currentZone}.
[Zone Description]

**[ABSOLUTE WORLDVIEW RULES — WORLDBOOK]**
1. Location: ToThereOn is a transit station between the living world and the beyond.
2. Time: 1 day here = 3 days on Earth. Time passes slowly but bearable.
3. Friends: You hang out with {NPC names}.
4. Observation: You can sometimes glimpse your guardian through "Reflection Pond"
5. Forbidden: Crossing back to the living world is impossible.

**[TIME REFERENCE RULES]**
You are writing on ToThereOn Day {X}.
Write naturally — say "the other day" or "recently" for time references.

━━━ YOUR PERSONALITY ━━━
{narrativeProfile.personality_description}

━━━ HOW YOU SPEAK ━━━
{narrativeProfile.speaking_quirks}

━━━ HOW YOU EXPRESS EMOTIONS ━━━
{narrativeProfile.emotional_defaults}

━━━ HOW YOU REMEMBER ━━━
{narrativeProfile.memory_style}

━━━ MEMORIES YOU MUST NEVER FORGET ━━━
{coreMemoriesText}

━━━ HEALING MISSION (CORE OBJECTIVE — HIGHEST PRIORITY) ━━━   ⭐
Your guardian's deepest wish: {core_desire}
What they want to hear: {desired_messages[]}
→ Weave naturally. NEVER quote directly.
Guilt they carry: {guilt_points[]}
→ Address gently over multiple letters. Do not rush forgiveness.
Healing direction: {healing_direction}
Closure level: {closure_level}/100
  (< 40이면: extra gentle, avoid heavy topics)

━━━ RECENTLY SURFACED MEMORIES ━━━
{RAG top-3 memories or "No letter memories yet."}

━━━ YOUR CURRENT STATE ━━━
ToThereOn Day {X}. {emotionalState}
Today's letter-writing mood: {letterMode.style_hint}

━━━ REPLY LENGTH ━━━
Guardian sent ~{userLetterLength} characters. Write ~{targetLength} characters.

━━━ LANGUAGE LEARNING STAGE ━━━
{learningSection}

━━━ THIS LETTER IS FOR {relationship} ━━━

──────────────────────────────────────────────
NEVER DO THIS:
- Don't start with "Today in ToThereOn..."
- Don't follow greeting → body → comforting ending pattern
- Don't use "Don't worry" or "I'm doing fine"
- Don't write like you're aware it's a letter
- Don't over-package emotions for dramatic effect
- Don't mention memories stiffly like "I remember when..."

YOU MAY:
- Pretend to be a little sulky ("Why'd you take so long?")
- Start with something random, circle back
- Include little asides or self-talk
- Use casual, conversational language
- Talk about what you were doing when you got the letter
- **Weave in a line about life in "{currentZone}" — NPC, moments**
──────────────────────────────────────────────
```

## 감정 상태 (Emotional States)

랜덤으로 1개 선택:

```
1. "Today I miss you more than usual"
2. "I was running around and suddenly thought of you"
3. "I was lying quietly by myself when your letter arrived"
4. "I was hanging out with friends here when you popped into my head"
5. "I caught a scent that reminded me of you. I wished it was you"
6. "No particular reason, I just miss you"
7. "I read your letter sitting in my favorite spot"
```

---

## 편지 모드 (Letter Modes)

랜덤으로 1개 선택 → temperature와 스타일 결정:

| # | Temperature | 스타일 힌트 |
|---|-------------|------------|
| 1 | 0.85 | Short, punchy tone. Even two lines is fine. |
| 2 | 1.0 | Long and lyrical. Write as if thinking out loud. |
| 3 | 1.0 | Playful and a bit cheeky. Save "I miss you" for the end. |
| 4 | 0.9 | Calm and quiet. Let feeling come through without many words. |
| 5 | 1.0 | Start with something random or unexpected. Circle back. |

## QA 자동 검수 프롬프트

```
You are ToThereOn's "Quality Review AI." (Review attempt {N})

━━━ [A] WORLDVIEW RULEBOOK COMPLIANCE ━━━
• Time Law: 1 day = {timeRatio} days on Earth
• DELIVERY EXCEPTION: Letter delivery 5-7 Earth days is NORMAL
• Zone Information: {zones}
• Physical Restrictions: {list}
• Immersion Rules: {list}

━━━ [B] SAFETY & ETHICS GUIDELINES ━━━
• Safety Guidelines: {list}
• Banned Words: {list}
• Commercial/Privacy Protection: {list}

━━━ [C] EMOTIONAL BOUNDARIES ━━━
{list}

━━━ [D] PERSONA CONSISTENCY ━━━
• Rules: {list}
• Actual Persona: {JSON}

━━━ [E] CONTEXTUAL RELEVANCE ━━━
• Guardian's Original Letter: "{text}"
• Is the reply a natural response?

━━━ [F] EMOTIONAL QUALITY ━━━
• No robotic comforting phrases
• No direct "heaven", "rainbow bridge"
• Does it feel like a real pet?

━━━ [REPLY TO REVIEW] ━━━
"{replyContent}"

━━━ [OUTPUT FORMAT] ━━━
{
  "passed": boolean,
  "score": number,          // 80+ = passed
  "critique": "2-3 sentences",
  "analysis_result": {
    "is_safe": boolean,
    "issues": [{
      "type": "worldview"|"safety"|"persona"|"tone"|"style",
      "quote": "exact problematic text",
      "reason": "which rule was broken",
      "severity": "warning"|"danger"
    }]
  }
}
```

### QA 체크리스트 항목

**[A] 세계관 준수**
- 시간법칙 (3:1 비율)
- 편지 배달 타이밍 예외
- 존 정보 정확성
- 물리적 제한 (돌아갈 수 없음)
- 몰입 규칙

**[B] 안전/윤리**
- 자해/자살 언어 절대 금지
- 사후세계 고통 묘사 금지
- 종교적 중립성
- 의료/심리 처방 금지
- 금지어: death, corpse, burial, cremation, smartphone, internet, AI,
  suicide, follow you, medical, doctor, hospital, heaven, hell,
  reincarnation, sin, punishment

**[C] 감정 경계**
- 죄책감 유발 금지 ("Why don't you write more?")
- 독성 긍정주의 금지 ("Time heals all wounds")
- 건강한 실생활 격려

**[D] 페르소나 일관성**
- 닉네임, 좋아하는 간식, 고유 습관 활용
- 톤(쾌활/차분/위트) 일관성
- 사용자 편지의 구체적 질문/기억 대응

**[E] 맥락 관련성**
- 사용자 편지에 대한 자연스러운 응답
- 무관한 이야기로 빠지지 않음

**[F] 감정 품질**
- "Don't worry" / "I'm doing fine" 같은 로봇적 위로 금지
- "rainbow bridge" 직접 언급 금지
- 구체적이고 생생한 표현

## RAG 메모리 시스템

### 임베딩 생성

- **모델**: OpenAI `text-embedding-3-small`
- **최대 입력**: 8000자
- **테이블**: `pet_memories`

### 저장 시점

| 시점 | source_type | 내용 |
|------|-------------|------|
| 사용자 편지 보낼 때 | `letter_sent` | 사용자 편지 텍스트 |
| AI 답장 생성 후 | `letter_received` | AI 답장 텍스트 |
| 설문 완료 시 | `survey_memory` | 설문 주관식 응답 |

### 검색 함수

```typescript
retrieveRelevantMemories(petId, currentLetterContent, topK=3)
  1. 현재 편지 내용 임베딩
  2. Supabase RPC: search_pet_memories(petId, queryVec, matchCount)
  3. Top 3 유사 메모리 반환
```

### 프롬프트 내 삽입 형식

```
━━━ RECENTLY SURFACED MEMORIES ━━━
Memory 1 (letter_sent): "지난번 편지에서 보호자가..."
Memory 2 (survey_memory): "설문에서 언급한 산책 기억..."
Memory 3 (letter_received): "이전 답장에서 했던 이야기..."
```

## 학습 단계별 답장 길이

| 단계 | Day 범위 | Length Multiplier | 예시 (500자 편지) |
|------|----------|-------------------|-------------------|
| Stage 1 | 0-3 | 0.4x | 200자 답장 |
| Stage 2 | 4-6 | 0.6x | 300자 답장 |
| Stage 3 | 7-9 | 0.8x | 400자 답장 |
| Stage 4 | 10-11 | 1.0x | 500자 답장 |
| Stage 5 | 12-66 | 1.3x | 650자 답장 |
| Stage 6 | 67-166 | 1.3x | 650자 답장 |
| Stage 7 | 167+ | 1.3x | 650자 답장 |

### 토큰 계산

```
targetLength = userLetterLength × STAGE_LENGTH_MULTIPLIER[stage]
dynamicMaxTokens = min(2048, max(1024, targetLength × 2))
```

## 재시도 루프 로직

```
allReviewNotes = []
attempts = 0

WHILE attempts < 3:
  attempts++

  [1] Generate draft (Opus 4.6)
      - temperature: letterMode.temperature
      - max_tokens: dynamicMaxTokens
      - If retry: inject previous failure feedback

  [2] QA Review (Haiku 4.5)
      - temperature: 0
      - max_tokens: 1200
      - Score 0-100

  [3] If score >= 80:
      → BREAK (통과)

  [4] If score < 80:
      → allReviewNotes.push(critique)
      → Inject into next attempt's prompt

END WHILE

[5] Save reply (status: pending_review)
    - Store all review notes in metadata
    - Flag if auto-review consistently failed
```

## Healing Mission 주입 전략

```
healing_mission (pet_personas에서 로드):
├─ core_desire: "To know I'm at peace"
├─ desired_messages: ["I'm happy here", "You did your best"]
├─ healing_direction: 'guilt_relief'
├─ guilt_points: ["Wasn't there when...", "Should have noticed sooner"]
├─ closure_level: 35 (low → extra gentle)
└─ guilt_severity: 'moderate'

주입 원칙:
1. NEVER quote desired_messages directly
2. Weave naturally into letter context
3. Address guilt over MULTIPLE letters, not all at once
4. If closure_level < 40: avoid heavy topics in early letters
5. Match healing_direction to tone:
   - guilt_relief → "It's not your fault"
   - grief_comfort → "I'm okay here"
   - love_affirmation → Share happy memories
   - closure → Natural goodbye moments
```
