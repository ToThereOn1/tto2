# 09. ToThereOn 서비스 작동구조 — 현황 스냅샷 (2026-03-07)

> 이전 문서(00~08)는 설계 기준 문서. 이 문서는 **현재 코드 기준 실제 작동 상태**를 기록.
> 두 문서 간 차이가 있을 경우 이 문서가 우선.

---

## 1. 전체 서비스 개요

ToThereOn은 무지개다리를 건넌 반려동물이 보호자와 재회하기 전까지 머무는 중계 세계에서
**매일 근황 피드**를 보내고, **편지에 답장**을 쓰는 감정 치유 SaaS 서비스입니다.

### 핵심 UX 루프

```
[보호자]
  │
  ├─ 1. 펫 등록 + Deep Remembrance 설문 (21문항)
  │       → AI가 펫 페르소나 생성 (12-Step Master Analyzer)
  │
  ├─ 2. 매일 펫 피드 수신 (CRON: Free 7일/Basic 3일)
  │       → 1인칭 저널 형식, 펫의 목소리로 ToThereOn 일상 전달
  │
  └─ 3. 편지 주고받기 (월 한도 내)
          → 보낸 후 7일 뒤 펫 답장 도착 (7단계 학습 글쓰기 진화)
```

## 2. 스테이지별 상세 작동 구조

### STAGE 1: 펫 등록
- **진입**: `/dashboard/pets/new` → `PetRegistrationForm`
- **서버사이드**: `pet_limit` 체크 → 초과 시 `UpgradeNudge` 표시
- **저장**: Supabase Storage (pet-photos 버킷) + `pets` 테이블 INSERT
- **초기값**: `persona_generated = false`
- **요금제 제한**:

| 티어 | 펫 제한 | 편지 한도 |
|------|---------|-----------|
| Free | 1마리 | 2통/월 |
| Basic ($9.99) | 1마리 | 4통/월 |
| Plus/Pro | Coming Soon | — |

---

### STAGE 2: Deep Remembrance 설문 (v2.2)
- **경로**: `/dashboard/pets/[petId]/remembrance`
- **문항**: 21문항, 5 Phase (영어 전용)
- **자동저장**: 5초마다 → `deep_remembrance_responses` (JSONB)
- **핵심 문항**:
  - **Q17** (듣고 싶은 말) → `healing_mission` 추출 — 최우선 분석 대상
  - **Q18** (전하고 싶은 말) → `guilt_points`, `closure_level` 추출
  - **Q04/Q06/Q12/Q13** → 전용 Multi-Choice 핸들러 처리

```
Phase 1: The Door    (Q01-Q03) — 첫 만남, 현관문 반응
Phase 2: The Living Room (Q04-Q10) — 일상, 놀이, 감정
Phase 3: The Quiet Room  (Q11-Q14) — 유대, 애정
Phase 4: The Letter  (Q15-Q19) — 추억, 소원, 편지 말투 ⭐ Q17 핵심
Phase 5: The Light   (Q20-Q21) — 일과, 준비
```

---

### STAGE 3: 페르소나 생성 (백그라운드, 비동기)
- **트리거**: `POST /api/deep-remembrance/complete` → `after()` 비동기
- **파이프라인**: `lib/analysis/master-analyzer.ts` 12단계

```
Step 1:  Breed Baseline 로드
Step 2:  Objective Scores (선택지 점수 집계)
Step 3:  Multi-Choice 패턴 분석 (Q04/Q06/Q12/Q13 전용 핸들러)
Step 4:  NLP 주관식 분석 — Claude Sonnet 4.6 (temp 0.2-0.3)
         └─ Q08,Q14,Q15,Q16 → personality/quirks/moments
            Q17+Q18 → healing_mission + guilt_points ⭐
Step 5:  Breed Correction (설문 70% + 품종 30% 블렌딩)
Step 6:  Cross-Validation (표준편차 > 25 → 중앙값 사용)
Step 7:  Big Five Mapping (energy/sociability/neuroticism/agreeableness/openness)
Step 8:  Emotional Profile 조립
Step 9:  Relationship Dynamics (attachment_style, family_role)
Step 10: Uniqueness 추출 (quirks, signature_moments, essence)
Step 11: Grief Context + Healing Mission ⭐
Step 12: Final Integration (confidence_score, data_quality)
```

- **출력**: `pet_personas` 테이블 UPSERT
  - `persona_analysis` (전체 분석 JSON)
  - `system_prompt` (500+ line 조립된 프롬프트)
  - `healing_mission` (JSONB) ⭐ 편지 답장의 핵심 목표
  - `analysis_version: '2.2'`
- **후속**: `pets.persona_generated = TRUE` → CRON 트리거 활성화

### STAGE 4: 펫 피드 이벤트 생성 (Daily CRON) — **v3.0 현재**

> ⚠️ 문서 04와 비교: v3.0에서 **3인칭 내레이터 → 1인칭 저널** 로 전환됨

#### CRON 플로우

```
GET /api/cron/daily-tothereon-check (Bearer 인증)
  ├─ 피드 주기 게이트 (Free: 7일, Basic: 3일)
  ├─ persona_generated = true 펫만 선별
  ├─ CHUNK_SIZE = 20마리 / 실행 (300초 제한)
  └─ 각 펫별 이벤트 생성
```

#### 이벤트 타입 결정 (우선순위 순)

| 우선순위 | 조건 | 이벤트 타입 |
|---------|------|------------|
| 1 | 편지 있음 + 168h 경과 (노출 가능) | `letter_response` |
| 2 | Day 0 | `arrival` |
| 3 | 학습 스케줄일 + stage != completed | `learning_observation / first_lesson / practice_session` |
| 4 | Day 7, 30, 100, 365 | `milestone` |
| 5 | 성격 기반 확률 | `npc_interaction / exploration / daily_routine` |

#### 학습 스케줄

| 속도 (IQ 기준) | 관찰 | 수업 | 연습 | 완료 |
|---------------|------|------|------|------|
| fast (≥75) | Day 2 | Day 3 | Day 4 | Day 5 |
| medium (≥50) | Day 3 | Day 5 | Day 7 | Day 8 |
| slow (<50) | Day 4 | Day 6 | Day 8 | Day 9 |

#### 현재 LLM 구조 (v3.0 — 1인칭 저널)

**System Prompt** (`lib/event-generator.ts:SYSTEM_PROMPT_BASE`):
```
핵심 페르소나: "You are a pet living in ToThereOn World."
3가지 규칙:
  RULE 1 — TELL WHAT HAPPENED (사건 중심)
  RULE 2 — SHOW, DON'T EXPLAIN YOUR FEELINGS (행동으로 감정 표현)
  RULE 3 — ONLY YOU DO IT THIS WAY (이 펫만의 순간)
+ YOUR VOICE 섹션: PersonaProfile.communication_style 주입
+ 언어별 금지어 섹션 (한/영/일)
길이: 2-3 paragraphs, 80-130 words
시점: 1인칭, 과거시제
```

**User Prompt** (`lib/prompt-builder.ts:buildStatusFeedPrompt`):
```
STORY CHAPTER (Day Arc: THE CROSSING ~ A LEGEND)
YOU ARE (personality_summary + core_traits + joy_triggers + daily_routines)
BEHAVIORAL TENDENCIES (dimensional scores → 설명 텍스트)
NARRATIVE THREAD ← 이전 3개 이벤트 연결 (MANDATORY)
  + OPENING BAN: 직전 이벤트 첫 문장 재사용 금지
TODAY'S SITUATION (narrative trigger + location + NPC)
  + NPC 첫 만남/재회 분기
MEMORY ANCHORS — PICK ONE AND USE IT (MANDATORY)
REFLECTION POOL RULE (warmth only)
FORBIDDEN WORDS (world-bible.ts 주입)
```

#### 검증 (validateOutput)
- 금지어 체크 (영/한/일 병렬 검사)
- 문장 수: 3-8 문장
- 단어 수: 최대 150 단어 (130 권장)
- 최소 길이: 100자
- 익명 동물 표현 금지 (named NPC만 허용)

## 5. 현재 알려진 구조적 한계 및 개선 가능 영역

### 5-A. 피드 콘텐츠 품질

| # | 항목 | 현황 | 리스크 |
|---|------|------|--------|
| 1 | **PersonaProfile 미활용 필드** | `letter_generation_guidelines`, `NarrativeData.voice_tone`, `secret_habit` 미주입 | 페르소나 몰입 한계 |
| 2 | **단기 메모리만 참조** | 최근 3이벤트만 내러티브 앵커로 사용 | 장기 연속성 단절 |
| 3 | **mood 고정** | `pet_status_events.mood = 'peaceful'` 하드코딩 | 감정 다양성 없음 |
| 4 | **milestone 이벤트 빈약** | Day 7/30/100/365에 특별 프롬프트 없음 | 기념일 임팩트 약함 |
| 5 | **학습 이벤트 내러티브 얕음** | observation/lesson/practice 타입만 분기, 특별 프롬프트 미완성 | 초기 Day 서사 단조 |
| 6 | **letter_response 게이트 168h** | 피드가 편지 답장 공개 전에 편지 내용 반영 불가 | 편지-피드 연결감 약함 |

### 5-B. 세계관 깊이

| # | 항목 | 현황 | 리스크 |
|---|------|------|--------|
| 7 | **가디언 배경 스토리 없음** | 이름/역할만 존재, origin story 없음 | NPC 만남 의미 약함 |
| 8 | **NPC 간 관계망 없음** | 각 NPC 독립 존재 | 세계의 사회성 빈약 |

| 10 | **존 전환 이벤트 없음** | Day 기반 자동 전환, 의식/이벤트 없음 | 성장 서사 단절 |
| 11 | **펫 간 우정 레벨 없음** | 60% 재선택만 있음, 우정 축적 미구현 | NPC 관계 평평함 |

### 5-C. 시스템 안정성

| # | 항목 | 현황 | 리스크 |
|---|------|------|--------|
| 12 | **validation 오탐** | word count 150 cap이 자연스러운 문장 자름 | 재시도 낭비 |
| 13 | **편지 빈도 미반영** | 오랜만의 편지가 피드/답장에 영향 없음 | 공백 감 없음 |

| 15 | **피드-편지 답장 연결 약함** | 피드에서 편지 인식 가능, 역방향은 제한적 | 두 채널 독립 서사 |
