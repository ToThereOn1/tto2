## 1. 현황 분석 — 코드 구조

### 답장 생성 흐름

```
보호자 편지 도착
  → reply-generator.ts:buildSystemPrompt()
      - random EMOTIONAL_STATES 선택
      - random LETTER_MODES 선택
      - HealingMission 주입 (core_desire, guilt_points 등)
      - buildLearningPromptSection() 주입 (단계별 글씨 지침)
      - PersonaProfile 주입 (coreTraits, joyTriggers 등)
  → USER MESSAGE: "Here's a letter from your ${relationship}:\n\n\"${letter.content}\""
  → Opus 4.6 생성 (max 600 tokens)
  → Haiku 리뷰 (score ≥ 80 통과, max 3회 재시도)
  → pending_review 상태로 저장 (current_tothereon_day 포함)
  → Admin 승인 → delivered
```

### 학습 단계 UIUX 흐름

```
letters DB: current_tothereon_day 저장 ✅ (reply-generator.ts:480)
  → /api/timeline/[petId]: current_tothereon_day 반환 ✅ (route.ts:188)
  → MailboxPage: matched?.current_tothereon_day ✅
  → LetterReadModal: getLearningStage(currentTothereonDay)
  → LetterPaper: stage prop 전달 → 폰트/흔들림 렌더링 ✅
```

**학습 단계 파이프라인 자체는 정상 작동.** 단, 구현 세부의 품질 문제가 있음.

## 2. 핵심 문제 진단

### 문제 A: 편지 상호작용이 구조적으로 부재 (가장 심각)

**현재 system prompt 구조:**
```
[캐릭터 페르소나] → [HealingMission] → [학습 단계 지침] → [월드 세계관]
→ [IMPORTANT] Weave in a line or two about life in ToThereOn World
→ USER MESSAGE: "Here's a letter from your guardian: "..."
```

**문제점:**
- 보호자 편지는 user message 한 줄로만 전달 — system prompt에 구조적 위치가 없음
- `[IMPORTANT] Weave in a line or two about life` → 세계관 일상이 주, 편지 답변이 부
- 실제 생성 패턴: 세계관 일상 서술 70% + 막연한 편지 언급 30%
- 보호자가 편지에서 한 구체적인 말(특정 단어/질문/기억)에 반응하는 구조가 없음
- random EMOTIONAL_STATES/LETTER_MODES가 편지 내용보다 생성 방향을 더 강하게 지배함

**예상 영향:** 편지를 읽은 보호자가 "내가 한 말에 답을 한 건지 잘 모르겠다"는 느낌

---

### 문제 B: 학습 단계 UIUX 실제 적용 격차

**작동하는 것:**
- DB에 `current_tothereon_day` 저장 ✅
- LetterPaper 7단계 폰트/흔들림 시스템 구현 ✅
- LetterReadModal에서 getLearningStage() 호출 ✅

**작동하지 않는 것 / 약한 것:**

1. **LLM 의존 오탈자**: `buildLearningPromptSection()`이 stage 1-2 오탈자를 지시하지만,
   Opus 4.6은 instruction을 정확히 따르지 않을 수 있어 — 오탈자가 없거나 어색하게 삽입됨

2. **Stage 1 시각 강도 부족**: Gaegu 폰트 + wobble은 구현되어 있지만
   흔들림 범위(`-3°~+3°, -1.5~+1.5px`)가 미미할 수 있음 — 실제 아이가 쓴 글씨처럼 보이지 않음

3. **구버전 편지 폴백**: `currentTothereonDay`가 null인 구버전 편지는
   LetterPaper 미적용 → 평범한 sans-serif div로 표시 — 학습 테마가 완전히 사라짐

4. **sent 편지는 폴백**: 보호자가 보낸 편지(sent type)는 의도적으로 plain div — 이는 정상

---

### 문제 C: 세계관 일상 vs 편지 답변 비율 역전

**현재 비율 (추정):** 세계관 일상 60-70% / 편지 답변 30-40%
**목표 비율:** 편지 답변 60-70% / 세계관 일상 30%

보호자 입장에서 편지를 보내는 이유: 펫이 "내 편지를 읽고 반응했다"는 경험 → 현재 미흡

## 4. 구체적 개선 방법

### 변경 1: 편지 내용을 System Prompt에도 주입 [1순위]

**적용 위치**: `lib/reply-generator.ts` → `buildSystemPrompt()` 함수 내부

**현재 문제**: 보호자 편지가 user message로만 전달되어 system prompt의 모든 지시보다 우선순위가 낮음

**개선 방향**: system prompt 끝에 편지 요약 섹션 추가

**OLD (현재 system prompt 마지막 부분):**
```
[IMPORTANT] Weave in a line or two about life in ToThereOn World —
a place they visited, someone they encountered, a small sensory moment.
```

**NEW (system prompt에 추가할 섹션):**
```
━━━ YOUR GUARDIAN'S LETTER ━━━
Your guardian wrote to you. Here is what they said:
"${guardianLetterContent}"

HOW TO RESPOND:
1. FIRST: Pick ONE specific thing from their letter — a word, a question, a memory they mentioned,
   a feeling they expressed. ECHO it directly in your reply (quote a phrase, or show it changed
   something you did today).
2. SECOND: Share 1-2 sentences of what you've been doing in ToThereOn World — only if it connects
   to what they said, OR if it shows you're okay.
3. The letter should feel like a REPLY, not a postcard.

LETTER RESPONSE PATTERN (mandatory):
"When you said [echo of their words], I [what it made you do/feel/notice]."
OR: "Your letter reached me just as I was [world activity]. The part about [their words] —
    I kept thinking about it."
```

**토큰 영향**: +30~50 (system prompt에 편지 원문 복제)
**기대 효과**: 편지 답변 비중 30% → 60%+

---

### 변경 2: RANDOM EMOTIONAL_STATE → 편지 기반 감정 파생 [1순위]

**적용 위치**: `lib/reply-generator.ts` → EMOTIONAL_STATES 선택 로직

**현재 문제**: random으로 선택된 EMOTIONAL_STATE(reflective, nostalgic 등)이 편지 내용과 무관하게 생성 방향을 지배

**개선 방향**: 편지 내용이 있으면 EMOTIONAL_STATE 섹션 자체를 제거하고,
편지 기반 감정 파생 지침으로 교체

**구현 방식:**
```typescript
// reply-generator.ts: buildSystemPrompt() 내부
const emotionalContext = guardianLetter
    ? `━━━ YOUR EMOTIONAL STATE ━━━
Your guardian wrote to you today. Let their letter set your tone —
don't choose an emotion before reading it. Let the emotion emerge from what they said.`
    : `━━━ YOUR EMOTIONAL STATE ━━━\n${emotionalState}`;
```

**토큰 영향**: ±0 (교체)
**기대 효과**: 감정 일관성 + 편지 반응성 향상

---

### 변경 3: 학습 단계 오탈자 Few-shot 예시 강화 [3순위]

**적용 위치**: `lib/learning-stage.ts` → `buildLearningPromptSection()` → stage 1, 2 섹션

**현재 문제**: "Include 2-3 intentional typos" 지시만 있고, 어떤 오탈자인지 예시가 없음
→ Opus 4.6이 어색한 오탈자 삽입 또는 무시

**개선 방향**: 각 단계에 구체적 오탈자 패턴 예시 추가

**예시 추가 (stage 1):**
```
Stage 1 typo patterns:
- Missing final letter: "toda" (today), "whe" (when), "som" (something)
- Letter swap: "nad" (and), "hte" (the), "fo" (of)
- Doubled letter: "tooo" (too), "aand" (and)
- Space missing: "Icame" (I came), "itwas" (it was)
Apply 2-3 of these naturally — not on every word, only where a young writer might slip.
```

**토큰 영향**: +20~30 per stage 1/2 편지
**기대 효과**: stage 1-2 오탈자의 자연스러운 적용률 향상

---

### 변경 4: Stage 1-2 Wobble 강도 상향 [4순위]

**적용 위치**: `components/letters/LetterPaper.tsx` → `applyWobble()` 함수

**현재:**
```typescript
const angle = Math.random() * 6 - 3;      // -3 to +3 degrees
const yOffset = Math.random() * 3 - 1.5;  // -1.5 to +1.5 px
```

**NEW (stage별 강도 차등화):**
```typescript
// stage 1-2: 더 강한 wobble
const angle = Math.random() * 10 - 5;     // -5 to +5 degrees
const yOffset = Math.random() * 5 - 2.5;  // -2.5 to +2.5 px
// stage 3-4: 현재 유지
// stage 5+: wobble 없음 (현재도 없음)
```

**구현**: `applyWobble(char, stageNumber)` 형태로 stage 파라미터 추가

**기대 효과**: stage 1-2 시각적 차별화 — 실제 어린 강아지/고양이가 쓴 글씨 느낌

---

### 변경 5: 구버전 편지 폴백 UI [5순위]

**적용 위치**: `components/timeline/LetterReadModal.tsx` 내 `learningStage === null` 분기

**현재**: plain div with prose-stone (sans-serif처럼 보임)

**기대 효과**: 구버전 편지도 손편지 느낌 유지

## 6. 검증 방법

개선 후 답장 품질 평가 기준:
```
✅ 편지에서 구체적인 단어/장면 1개 이상 echo 됨
✅ 그 echo가 펫의 행동/감정에 연결됨
✅ 세계관 일상은 1-2문장 이내
✅ 보호자가 읽었을 때 "내 편지를 읽었구나" 느낌
```

평가 방식: Admin 패널에서 답장 3-5통 수동 리뷰 (LLM-as-Judge 점수보다 정성 평가 우선)
