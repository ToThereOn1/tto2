# 02. Deep Remembrance 설문 시스템

## 개요

펫 등록 후 진행되는 21문항 설문. 5단계 Phase로 구성되며, 사용자의 응답을 기반으로
펫의 성격 프로필(페르소나)을 생성하는 핵심 데이터 수집 단계입니다.

**버전**: v2.2 (영어 전용, 21문항, 5 Phase)
**이전 버전**: v1.0 (한영 혼합, 25문항, 6 Section)

---

## Phase 구조

| Phase | 이름 | 문항 | 주제 |
|-------|------|------|------|
| P1 | The Door | Q01-Q03 | 첫 만남, 현관문 반응 |
| P2 | The Living Room | Q04-Q10 | 일상 행동, 놀이, 감정 |
| P3 | The Quiet Room | Q11-Q14 | 깊은 유대, 애정 표현 |
| P4 | The Letter | Q15-Q19 | 추억, 소원, 편지 말투 |
| P5 | The Light | Q20-Q21 | 일과, 준비 확인 |

## 전체 문항 상세 (21개)

### Phase 1: THE DOOR (Q01-Q03)

#### Q01 — 첫 만남
- **질문**: "Do you remember the day you first met [Name]? How did you two find each other?"
- **타입**: single_choice
- **측정**: origin_story
- **선택지**:
  - family_brought — "My family brought them home"
  - rescue — "I rescued/adopted them"
  - gift — "They were a gift"
  - pet_store — "From a pet store/breeder"
  - born_family — "Born into our family"
  - dont_remember — "I don't remember clearly"
- **Other**: 허용

#### Q02 — 현관문 반응
- **질문**: "When you came home and opened the door, what did [Name] usually do?"
- **타입**: single_choice
- **측정**: social_energy(0.6), affection_style
- **선택지 + 점수**:

| 값 | 점수 | 설명 |
|----|------|------|
| ran_crazy | 95 | 미친듯이 달려옴 |
| wagging_tail | 80 | 꼬리 흔들며 맞이 |
| strolled | 60 | 느긋하게 다가옴 |
| lifted_head | 45 | 고개만 듦 |
| pretend_no_care | 55 | 무관심한 척 |
| tail_giveaway | 50 | 꼬리가 본심을 드러냄 |
| depended | 60 | 기분에 따라 다름 |

#### Q03 — 낯선 사람 반응
- **질문**: "When someone new came to your home, how did [Name] usually react?"
- **타입**: single_choice
- **측정**: social_energy(0.4), emotional_resilience
- **선택지 + 점수**:

| 값 | 점수 | 설명 |
|----|------|------|

### Phase 2: THE LIVING ROOM (Q04-Q10)

#### Q04 — 혼자 있을 때 ⭐ MULTI-CHOICE
- **질문**: "When you left the house and [Name] was home alone, what usually happened?"
- **타입**: multiple_choice
- **측정**: separation_anxiety, independence, coping_style
- **선택지**:

| 값 | 태그 | 의미 |
|----|------|------|
| followed | anxiety | 따라다님 |
| waited_door | attachment | 문 앞에서 기다림 |
| chewed | externalizer | 물건 씹음 |
| barked_cried | externalizer | 짖거나 울음 |
| slept_quietly | independence | 조용히 잠 |
| played_alone | independence | 혼자 놀음 |
| stopped_eating | internalizer | 밥을 안 먹음 |
| not_sure | neutral | 모르겠음 |

**분석 로직 (handleQ04_SeparationAnxiety)**:
```
anxiety_count = followed + waited_door + barked_cried + stopped_eating
independence_count = slept_quietly + played_alone
meaningful_total = anxiety_count + independence_count

anxiety_score = (anxiety_count / meaningful_total) × 100
independence_score = (independence_count / meaningful_total) × 100

coping_style:
  - anxiety > independence → anxious_attachment
  - independence > anxiety → self_sufficient
  - 비슷하면 → adaptive
```

#### Q05 — 새로운 것 반응

- **타입**: single_choice

#### Q06 — 놀이 스타일 ⭐ MULTI-CHOICE
- **질문**: "What kind of play made [Name] the happiest?"
- **타입**: multiple_choice
- **측정**: playfulness_intensity, play_style
- **선택지**:

| 값 | 유형 | 의미 |
|----|------|------|
| fetching | active_play | 던지기 놀이 |
| chasing | chase_play | 추격 놀이 |
| hide_seek | social_play | 숨바꼭질 |
| destroying | solo_play | 파괴 놀이 |
| wrestling | contact_play | 격투 놀이 |
| exploring | explorer_play | 탐험 |
| next_to_me | low_play | 옆에 앉기 |

**분석 로직 (handleQ06_PlayStyle)**:
```
active_items = fetching, chasing, hide_seek, destroying, wrestling, exploring
low_items = next_to_me

3+ active → intensity 85 (high_energy_diverse)
2 active → intensity 65 (moderate_active)
1 active → intensity 45 (selective_player)
only low → intensity 20 (companionship_oriented)
else → intensity 30 (low_key)
```

- **선택지**:

| 값 | 유형 | 의미 |
|----|------|------|
| came_to_hide | dependent | 보호자에게 숨음 |
| found_corner | internalizer | 구석에 숨음 |
| trembled | physiological | 떨림 |
| barked_cried | externalizer | 짖거나 울음 |
| chewed | externalizer | 물건 씹음 |
| stopped_eating | internalizer | 밥 안 먹음 |

**분석 로직 (handleQ13_StressResponses)**:

## UI 컴포넌트 상세

### DeepRemembranceSurvey.tsx

#### 핵심 기능
1. **질문 로드**: `survey_questions` 테이블에서 `is_active=true, version=2` 필터
2. **구조화된 응답**: answer_type별 분류 (single_choice, multiple_choice, text, other)
3. **5초 자동저장**: `deep_remembrance_responses` 테이블에 중간 저장
4. **Phase 전환**: 다음 문항의 phase가 다르면 PhaseTransition 모달 표시
5. **자기 복구**: 세션 ID 유실 시 DB에서 복구 또는 긴급 생성

#### 응답 데이터 구조

```typescript
// 단일 선택
{ answer_type: 'single_choice', question_key: 'Q02',
  selected_choice: { label, value, score: 95, dimension: 'social_energy' } }

// 다중 선택
{ answer_type: 'multiple_choice', question_key: 'Q04',
  selected_choices: [{ label, value, score, dimension }, ...] }

// 텍스트
{ answer_type: 'text', question_key: 'Q08', text_answer: "..." }

// 기타(Other)
{ answer_type: 'other', question_key: 'Q01',
  selected_choice: { label, value }, other_text: "..." }
```

### PhaseTransition.tsx

- 전체화면 모달 오버레이 (z-100)
- Aurora blob 애니메이션 (3개 색상 블롭)
- Phase 이름 그라디언트 텍스트 (sky-200 → blue-100 → indigo-200)
- 펫 이름 개인화된 인트로 텍스트
- Continue 버튼 → 클릭 시 다음 문항으로

## 설문 완료 플로우

```
[1] 마지막 답변 검증
    └─ 필수 질문 미응답 시 에러

[2] 세션 복구 (Self-Healing Safety Net)
    ├─ responseId가 유효하지 않으면?
    ├─ DB에서 기존 세션 조회 → 복구
    └─ 없으면 → 긴급 세션 생성 (전체 응답 포함)

[3] 응답 완료 마킹
    └─ UPDATE deep_remembrance_responses SET
         completion_percentage = 100,
         completed_at = now()

[4] 펫 상태 업데이트
    └─ UPDATE pets SET persona_generated = true

[5] 백그라운드 처리 트리거
    └─ POST /api/deep-remembrance/complete
         { petId, responseId }
    └─ 즉시 반환 (비동기 처리)

[6] 완료 페이지로 이동
    └─ /dashboard/pets/{petId}/remembrance/complete
    └─ "We're looking for [Name] now..."
```
