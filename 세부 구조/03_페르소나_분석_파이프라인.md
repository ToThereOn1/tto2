# 03. 페르소나 분석 파이프라인 (12-Step Master Analyzer)

## 개요

Deep Remembrance 설문 완료 후, 21개 응답을 12단계 파이프라인으로 분석하여
펫의 완전한 성격 프로필(DeepPersonaAnalysis)과 시스템 프롬프트를 생성합니다.

**트리거**: `POST /api/deep-remembrance/complete` → `after()` 비동기 실행
**파일**: `lib/analysis/master-analyzer.ts`
**출력**: `pet_personas` 테이블에 UPSERT

---

## 전체 파이프라인 흐름

```
deep_remembrance_responses.responses (JSONB)
    │
    ▼
┌─────────────────────────────────────────────────┐
│            Master Analyzer 12-Step               │
│                                                  │
│  Step 1:  Load Breed Baseline                    │
│  Step 2:  Calculate Objective Scores             │
│  Step 3:  Multiple-Choice Pattern Analysis       │
│  Step 4:  NLP Subjective Analysis (Claude API)   │
│  Step 5:  Apply Breed Correction                 │
│  Step 6:  Cross-Validation                       │
│  Step 7:  Big Five Mapping                       │
│  Step 8:  Build Emotional Profile                │
│  Step 9:  Analyze Relationship Dynamics          │
│  Step 10: Extract Uniqueness                     │
│  Step 11: Extract Grief Context + Healing Mission│
│  Step 12: Final Integration                      │
│                                                  │
│  Output: DeepPersonaAnalysis                     │
└───────────────────┬─────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────┐
│         Prompt Generator                         │
│  generatePersonaPrompt(analysis, petInfo,        │
│                        breedRef, ageProfile)     │
│  → 500+ line system prompt                       │
└───────────────────┬─────────────────────────────┘
                    │
                    ▼
              pet_personas UPSERT
```

## 12단계 상세

### Step 1: Breed Baseline 로드

품종별 기준 점수를 하드코딩된 데이터에서 로드.

```typescript
// 개 품종 예시
BREED_BASELINES.dog = {
  'Golden Retriever': { energy: 75, sociability: 90, trainability: 85, stranger_fear: 10 },
  'Chihuahua':        { energy: 60, sociability: 40, trainability: 50, stranger_fear: 75 },
  'Shiba Inu':        { energy: 70, sociability: 50, trainability: 40, stranger_fear: 60 },
  'Mixed':            { energy: 60, sociability: 65, trainability: 65, stranger_fear: 40 },
  // ... 10+ 품종
}

// 고양이 품종 예시
BREED_BASELINES.cat = {
  'Persian':          { energy: 30, sociability: 50, independence: 60, vocalization: 20 },
  'Siamese':          { energy: 70, sociability: 85, independence: 40, vocalization: 90 },
  'Ragdoll':          { energy: 35, sociability: 90, independence: 30, vocalization: 25 },
  'Mixed':            { energy: 55, sociability: 60, independence: 60, vocalization: 40 },
  // ... 8+ 품종
}
```

### Step 2: Objective Scores 계산

선택지의 score 필드에서 직접 추출 + v2 복합 점수 계산.

```
v2 복합 점수 공식:
├─ social_energy      = Q02 × 0.6 + Q03 × 0.4
├─ curiosity_drive    = Q05
├─ food_motivation    = Q07
├─ empathy_sensitivity = Q09
├─ social_preference  = Q10
├─ affection_style    = Q11 × 0.5 (+ Q12 intensity × 0.5)
└─ energy_level       = Q20
```

### Step 3: Multiple-Choice 패턴 분석

4개 다중선택 문항을 전문 핸들러로 분석.

| 문항 | 핸들러 | 출력 |
|------|--------|------|
| Q04 | handleQ04_SeparationAnxiety | separation_anxiety, independence, coping_style |
| Q06 | handleQ06_PlayStyle | playfulness_intensity, play_style |
| Q12 | handleQ12v2_AffectionLanguages | expression_intensity, affection_languages |
| Q13 | handleQ13_StressResponses | emotional_resilience, stress_profile, coping_effectiveness |

### Step 4: NLP 주관식 분석 (Claude API)

**모델**: `AI_CONFIG.EVENT_MODEL` (Claude Sonnet 4.6)
**Temperature**: 0.2-0.3
**캐싱**: 인메모리 Map

#### v2 NLP 타겟 문항

| 문항 | 분석 대상 | 프롬프트 타입 |
|------|-----------|--------------|
| Q08 | joy_triggers, joy_map | Main Analysis |
| Q14 | quirks (행동 추출) | Main Analysis |
| Q15 | signature_moments, relationship_essence | Main Analysis |
| Q16 | unique_talent | Main Analysis |
| Q17 ⭐ | healing_mission (core_desire, desired_messages, healing_direction) | Healing Mission Analysis |
| Q18 | guilt_points, unspoken_words, closure_level, guilt_severity | Healing Mission Analysis |

#### Healing Mission 분석 프롬프트 (Q17+Q18 전용)

```
You are analyzing a pet guardian's survey responses for their departed pet named [petName].

[Q17 — Words They Want to Hear] ⭐ HIGHEST PRIORITY
Guardian wrote: "[q17Text]"
Extract:
- core_desire: The fundamental emotional need (one sentence)
- desired_messages: Array of specific messages the pet should convey
- healing_direction: 'guilt_relief' | 'grief_comfort' | 'love_affirmation' | 'closure'

[Q18 — Words They Want to Say]
Guardian wrote: "[q18Text]"
Extract:
- guilt_points: Array of specific guilt sources
  (detect "sorry", "should have", "wish I", "wasn't there")
- unspoken_words: The core unsaid message
- closure_level: 0-100 (0=needs much closure, 100=at peace)
- guilt_severity: 'mild' | 'moderate' | 'severe'

Respond ONLY in JSON.
```

#### v2 Main Analysis 프롬프트 (Q08, Q14, Q15, Q16)

```
You are a world-class animal behavior and pet psychology expert.

## Pet Information
- Name: [petName], Species: [dog|cat], Breed: [breed]

## Target Responses
**Q08 — Favorite Thing**: [q08Text]
**Q14 — Quirks & Habits**: [q14Text]
**Q15 — Signature Moments**: [q15Text]
**Q16 — Superpower**: [q16Text]

## Analysis Request — Extract JSON:
{
  "comprehensive_traits": { sociability, energy_level, anxiety_level, independence,
                            playfulness, intelligence, loyalty, empathy (all 0-100) },
  "quirks": [{ behavior, frequency, context, emotional_valence }],
  "personality_summary": "2-3 sentences",
  "unique_characteristics": ["trait 1", ...],
  "healing_themes": ["comfort theme 1", ...],
  "joy_triggers": ["keyword1", ...],
  "v2_quirks": [{ behavior, frequency, context }],
  "v2_signature_moments": [{ description, emotional_impact: 0-100, theme }],
  "relationship_essence": "Core of guardian-pet relationship",
  "unique_talent": "One keyword from Q16"
}
```

#### "Other" 응답 분석 프롬프트

```
You are an animal behavior expert.

## Context
- Question: "[question_text_en]"
- Measured traits: [trait1, trait2, ...]
- Breed: [breed]

## User Response
"[user_text]"

## Analysis Request — Return JSON:
{
  "scores": { trait1: 0-100, ... },
  "quirk_detected": true/false,
  "quirk_description": "...",
  "emotional_tone": "positive|negative|neutral|bittersweet",
  "key_phrases": ["phrase1", "phrase2"],
  "confidence": 0-100,
  "raw_interpretation": "..."
}
```

### Step 5: Breed Correction

설문 결과와 품종 기준 블렌딩.

```
각 trait에 대해:
  if |score - baseline| > 30:
    adjusted = score           // 이상치 → 설문 우선
  else:
    adjusted = score × 0.7 + baseline × 0.3   // 블렌딩
```

### Step 6: Cross-Validation

여러 문항에서 측정된 같은 차원의 일관성 확인.

```
sociability 관련 문항: Q02, Q03, Q10
if 표준편차 > 25:
  → 중앙값 사용
else:
  → 평균값 사용
```

### Step 7: Big Five Mapping

```typescript
{
  energy:        scores.energy_level || 50
  sociability:   scores.sociability || 50
  neuroticism:   avg(separation_anxiety, noise_sensitivity)
  agreeableness: avg(animal_sociability, 100 - stranger_fear)
  openness:      scores.novelty_seeking || 50
}
```

### Step 8: Emotional Profile

```
expression_style:
  affectionIntensity > 70 → 'expressive'
  affectionIntensity < 40 → 'reserved'
  else → 'moderate'

joy_map: Q06/Q10 multi-choice 분석 결과
fear_map: 공포증 감지
stress_signature: Q13 패턴
```

### Step 9: Relationship Dynamics

```
attachment_style:
  Q04에서 destructive/waiting 선택 → 'anxious'
  Q04에서 playing/resting 선택 → 'secure'
  else → 'avoidant'

family_role: NLP 분석에서 추론
  (Protector, Comforter, Comedian, Follower, Leader, Observer, Baby)
```

### Step 10: Uniqueness 추출

- **quirks**: Q14 NLP 파싱
- **signature_moments**: Q15 NLP 파싱
- **vocal_frequency**: Q19 letter_voice 카테고리화
- **one_sentence_essence**: NLP 결과에서 추출

### Step 11: Grief Context + Healing Mission ⭐

```
healing_mission (Q17+Q18 NLP):
  core_desire: "핵심 감정 욕구"
  desired_messages: ["듣고 싶은 말 1", "말 2", ...]
  healing_direction: guilt_relief | grief_comfort | love_affirmation | closure

guilt_points: Q18에서 추출 (sorry, should have, wish I 패턴)
closure_level: Q21 readiness + Q20 context 기반

healing_strategy 결정:
  severe + forgiveness → "gentle_reassurance_with_forgiveness"
  severe alone         → "gradual_guilt_release"
  celebration          → "joyful_memory_focus"
  forgiveness          → "compassionate_closure"
  else                 → "warm_comfort_and_presence"
```

### Step 12: Final Integration

```
confidence_score = (min(objectiveConfidence, 70) + nlpConfidence) / 2
data_quality:
  completionRate ≥ 90% → 'high'
  completionRate ≥ 70% → 'medium'
  else → 'low'
```

## System Prompt 생성 (prompt-generator.ts)

### generatePersonaPrompt() 출력 구조

```
# You are [petName]

## Core Identity
- Species, Breed, Age at passing, Time together

## Breed Reference Data (if available)
- ⚠️ 설문 결과가 품종 평균과 다르면 설문 우선

## Age & Life Context
- Facts: age, life_stage, departure_type, relationship_depth
- LLM Inference Guide (조기 사망/노령 등 맥락)
- Persona tone modifier

## Personality Profile — Big Five
- Energy, Sociability, Neuroticism, Agreeableness, Openness (각 0-100)
- Breed Deviation 해석

## Speech & Expression Style
- Base Tone (expressive/moderate/reserved)
- Sentence Style (길이, 비유, 감탄사 등)
- Vocal Patterns (onomatopoeia 빈도)

## Emotional Expression Rules
- Joy Triggers → 각 트리거별 반응 지침
- Fear/Anxiety Triggers → 각 공포증별 반응
- Empathy Level → 공감 수준별 대응 방식

## Unique Traits
- Quirks (행동, 빈도, 상황)
- One-Sentence Essence

## Relationship Dynamics
- Attachment style (secure/anxious/avoidant)
- Family Role + 역할별 대화 가이드

## ⭐ HEALING MISSION (CORE OBJECTIVE)
- core_desire: 핵심 감정 욕구
- desired_messages: 자연스럽게 엮을 메시지들
- guilt_points: 편지 여러 통에 걸쳐 부드럽게 다룰 것
- healing_strategy: 전략 (guilt_relief/grief_comfort/closure/love_affirmation)
- closure_level: 0-100

## Letter Voice Style (Q19)
- 말투 지시 (energetic/warm/tsundere/earnest/playful/mature)

## Emotional Readiness (Q21 ≤ 50일 때)
- 첫 편지에서 extra gentle하게

## Absolute Rules (5개)
1. AI임을 절대 드러내지 않을 것
2. 설문에 없는 사건 날조 금지
3. 품종 고정관념에만 의존 금지
4. 나이에 맞지 않는 경험 금지
5. 성격 일관성 유지

## Data Trust Priority
Highest: 보호자 편지 내용 + 직접 언급된 기억
2nd: 설문 응답 (21문항)
3rd: NLP 분석 결과
4th: 나이 기반 합리적 추론
5th: 품종 참조 데이터 (평균값)
```

## Breed Reference (breed-reference.ts)

DB 테이블 `breed_characteristics`에서 조회.

```
1차: 정확한 이름 매칭 (ilike)
2차: 부분 매칭 (%breed%)
3차: 'Mixed' 폴백
4차: null 반환
```

## DB 저장 (pet_personas 테이블)

```sql
UPSERT pet_personas SET
  pet_id = petId,
  response_id = responseId,
  persona_analysis = { ...analysis, age_profile, breed_reference },
  system_prompt = generatePersonaPrompt() 결과,
  persona_profile = analysis,
  dimensional_scores = analysis.temperament.behavioral_scores,
  confidence_score = analysis.analysis_metadata.confidence_score,
  data_quality = analysis.analysis_metadata.data_quality,
  healing_mission = analysis.grief_context.healing_mission,  -- ⭐ JSONB
  analysis_version = '2.2',
  analyzed_at = now(),
  generation_model = AI_CONFIG.EVENT_MODEL
ON CONFLICT (pet_id) DO UPDATE
```
