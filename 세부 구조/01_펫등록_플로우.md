# 01. 펫 등록 플로우 (Pet Registration)

## 개요

사용자가 반려동물을 ToThereOn 세계에 등록하는 첫 번째 단계.
등록 완료 후 Deep Remembrance 설문으로 연결됩니다.

---

## 진입 경로

```
/dashboard/pets/new → PetRegistrationForm 렌더링
```

### 서버사이드 사전 체크 (`app/dashboard/pets/new/page.tsx`)

```
1. 사용자 인증 확인 → 미인증 시 /login 리다이렉트
2. 현재 등록 펫 수 조회: SELECT count(*) FROM pets WHERE user_id = ?
3. 사용자 펫 제한 조회: SELECT pet_limit, max_pets_allowed FROM users WHERE id = ?
4. petLimit = pet_limit ?? max_pets_allowed ?? 1
5. maxPetsReached = (현재 펫 수 >= petLimit)
```

**제한 초과 시**: `<UpgradeNudge>` 컴포넌트 표시
- 메시지: "Bring your other companion to ToThereOn World"
- CTA: "See Plus Plan" → `/pricing`

## 등록 폼 필드

### 필수 필드

| 필드 | 타입 | 검증 규칙 |
|------|------|-----------|
| `name` | text | 비어있으면 안됨 |
| `passed_date` | month (YYYY-MM) | 필수, birth_date 이후여야 함 |
| `species` | dropdown | dog, cat, rabbit, bird, hamster, other |
| `gender` | radio | male 또는 female |
| `photos` | file upload | 최소 1장, 최대 5장 |

### 선택 필드

| 필드 | 타입 | 비고 |
|------|------|------|
| `birth_date` | month (YYYY-MM) | passed_date 이전이어야 함 |
| `breed` | text | 품종명 (null 허용) |
| `weight_kg` | number | 소수점 허용 (step=0.1) |
| `relationship` | multi-select buttons | mom, dad, friend, sister, brother, guardian, other |

## 사진 업로드 로직

### 제약 조건
- 허용 형식: JPEG, PNG, WebP만
- 최대 파일 크기: 5MB / 장
- 최대 총 사진 수: 5장

### 업로드 플로우

```
1. 파일 선택 → 타입/크기 검증
2. 미리보기 URL 생성 (URL.createObjectURL)
3. 제출 시 각 파일을 Supabase Storage에 업로드
   - 버킷: pet-photos
   - 경로: {userId}/{timestamp}_{randomString}.{ext}
4. 공개 URL 배열 반환 → pets.photos에 저장
```

## 폼 제출 플로우 (handleSubmit)

```
[1] 클라이언트 검증
    ├─ name 비어있는지
    ├─ passed_date 존재하는지
    ├─ species 선택했는지
    ├─ gender 선택했는지
    ├─ photos 최소 1장인지
    └─ birth_date < passed_date (둘 다 있을 때)

[2] 인증 확인
    └─ supabase.auth.getUser() → 없으면 /login

[3] 펫 제한 확인
    └─ maxPetsReached → 에러 표시 + 차단

[4] 사진 업로드
    └─ 각 파일 → Supabase Storage → public URL 수집

[5] DB 레코드 생성
    └─ supabase.from('pets').insert({
         user_id, name, birth_date, passed_date,
         species, breed, weight_kg, gender,
         relationship, photos: [url1, url2, ...]
       }).select().single()

[6] 성공 처리
    ├─ onSuccess(pet.id) 콜백 호출 (있으면)
    └─ 또는 /dashboard로 이동
```

## API 라우트: POST /api/pets

### 서버사이드 펫 제한 체크

```typescript
// 1. 사용자 구독 데이터 조회
const userData = await supabase.from('users')
    .select('pet_limit, max_pets_allowed, subscription_tier')
    .eq('id', user.id).single()

// 2. 티어별 펫 제한 결정
const tier = (userData.subscription_tier || 'free').toLowerCase()
const petLimit = PLAN_PET_LIMITS[tier] || 1

// 3. 현재 펫 수 카운트
const { count } = await supabase.from('pets')
    .select('*', { count: 'exact', head: true })
    .eq('user_id', user.id)

// 4. 제한 초과 시 → 403 PET_LIMIT_REACHED
```

### 응답 코드

| 코드 | 의미 |
|------|------|
| 200 | 성공 (pet 객체 반환) |
| 401 | 미인증 |
| 403 | PET_LIMIT_REACHED |
| 404 | 사용자 DB 레코드 없음 |
| 500 | DB 삽입 실패 |

## 영향 관계 맵

```
PetRegistrationForm
    ├─→ Supabase Storage (pet-photos 버킷)
    ├─→ pets 테이블 INSERT
    │     ├─→ persona_generated = false (초기값)
    │     └─→ photos = [url1, url2, ...]
    ├─→ users.pet_limit 체크 (구독 기반)
    └─→ 다음 단계: DeepRemembranceSurvey

PLAN_PET_LIMITS (lib/constants/plans.ts)
    └─→ POST /api/pets에서 참조
    └─→ /dashboard/pets/new 서버사이드에서 참조

UpgradeNudge 컴포넌트
    └─→ 제한 초과 시 /pricing 유도
```
